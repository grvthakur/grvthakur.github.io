01 / About

Engineer of
reliable data.

I'm a Data Engineer at Tata Consultancy Services (TCS) with 5+ years of experience building scalable data infrastructure. I hold dual Databricks Certified Data Engineer credentials — Professional (Mar 2026) & Associate (Oct 2025).

My core work: designing ETL/ELT pipelines that ingest from REST APIs and heterogeneous sources, transform with Apache Spark / PySpark, land in Delta Lake under Medallion Architecture, and serve clean, validated data to downstream BI and ML consumers.

Beyond the lakehouse I have deep roots in MS SQL Server, T-SQL performance tuning, and data quality frameworks — the unglamorous plumbing that makes production pipelines trustworthy.

01

Measure data quality

Don't claim a pipeline is clean until the DQ framework says so. Checks are first-class, not afterthoughts.

02

Schema as contract

Downstream consumers depend on structural guarantees. Schema drift is a production incident waiting to happen.

03

Idempotent by default

Every pipeline run should be safe to re-run. Delta Lake + ACID compliance makes this non-negotiable.

04

Config over hardcode

Connection strings, thresholds, and schedules live in config — not buried in transformation logic.
